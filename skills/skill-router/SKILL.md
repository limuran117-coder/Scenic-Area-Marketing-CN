# skill-router

> 统一技能路由 | 始终处于激活状态

## 核心理念

**用户不需要记住触发词。** 只需要用自然语言表达意图，路由器帮你找到最合适的技能。

## 工作流程

```
用户消息
    ↓
1️⃣ 语义意图识别（keyword → skill映射）
    ↓
2️⃣ 如果没匹配到 → 主动问用户是否需要某项技能
    ↓
3️⃣ 如果有明确需求 → 直接调用对应技能
    ↓
4️⃣ 每次识别都更新使用计数（高频激活）
```

## 主动推送机制

### 当用户提到以下「弱信号」时，主动询问：
- "忘了" / "不知道怎么做" / "有什么可以"
  → 推送技能清单
- "最近好像没跑" / "没收到"
  → 检查对应cron上次执行时间
- 任何竞品相关但没说具体
  → "你想分析哪个竞品？"

### 每周日Wiki LINT后推送：
- "本周新增了N个未归档脚本，要运行一次代码库漂移检查吗？"

## 语义匹配规则

不是严格精确匹配，模糊匹配也触发：
```
"抖音今天怎么样" → 匹配"抖音" → 抖音指数日报
"帮我查查对手" → 匹配"竞品" → 竞品关键词深度分析
"做个周报" → 匹配"周报" → 周度客流营收洞察
"能不能帮我xxx" → 模糊 → 主动问"你是想...？"
```

## 记忆机制

每次成功匹配，在 memory/skill-usage.md 记录：
```markdown
## YYYY-MM-DD
- "抖音数据" → 抖音指数日报 ✅
- "竞品分析" → 竞品关键词深度分析 ✅
- "怎么做" → 推送技能清单 ✅
```

高频使用的技能排名靠前。

## 在本项目的特殊规则

**电影小镇景区营销上下文**（自动注入）：
- 用户问"景区数据" → 默认理解为电影小镇

- 用户问"今天" → 自动加上日期上下文
