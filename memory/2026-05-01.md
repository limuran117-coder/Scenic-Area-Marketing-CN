### 已修复的Bug（本次会话发现的）

**countdown.js v2 存在4个严重Bug：**

1. **`getElapsedText()` d借位错误**（严重）
   - 错误：`const prevMonth = new Date(now.getFullYear(), now.getMonth() - 1, 0)`
   - 正确：`const prevMonth = new Date(start.getFullYear(), start.getMonth(), 0)`
   - 借位应该基于 startDate 的月份，不是 now 的月份

2. **`getElapsedText()` 无防御**（中等）
   - startDate 为 null/undefined/空字符串 时会创建 `Invalid Date`
   - `isNaN(start.getTime())` 判断后兜底处理

3. **`getElapsedText()` 引用错误**（严重）
   - `cd.getDiff(targetDate, now)` → 应为 `getDiff(targetDate, now)`
   - `cd` 未定义，会直接抛错

4. **`getMainCountdown()` 无防御**（中等）
   - targetDate 为 null/空/无效时，`new Date(null)` 返回 `Wed Dec 31 1969` 或 Invalid Date
   - 添加字符串空值判断 + `isNaN(target.getTime())` 兜底

**index.js SHARE_THEME 缺少 titleColor：**
- `death.titleColor` 缺 → 补 `'#E0E0EC'`
- `pet_birthday.titleColor` 缺 → 补 `'#4A2810'`

**index.wxml 缺少 shareCanvas：**
- 需要 `<canvas canvas-id="shareCanvas" style="width:300px;height:420px;position:absolute;left:-9999px;top:-9999px;"></canvas>`
- 否则 `shareCard()` 调用 `wx.createCanvasContext('shareCanvas')` 失败

**detail.wxml canvas高度不一致：**
