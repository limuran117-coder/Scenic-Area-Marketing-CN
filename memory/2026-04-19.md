## 飞书卡片格式标准化（12:00-15:00）

### 两种渲染方式对比

| 方式 | 实现 | 标题字体 | 优点 | 缺点 |
|------|------|---------|------|------|
| OpenClaw card | `message(action=send, card=...)` | OpenClaw渲染为大图，标题更大 | 视觉效果好 | API兼容性问题，图片嵌入方式 |
| Python直调API | `scripts/send_feishu_card.py` | 飞书原生markdown，标题有限制 | 稳定可靠 | 标题字号无法调整 |

**根因分析**：
- OpenClaw的card参数先把卡片渲染成图片，再用`tag:"img"`内嵌（message_id: om_x100b51711067bca8c2e34d2af92b3de）
- Python直调API是原生JSON，飞书自己渲染，markdown不支持字号调节
- 两种方式的`img`标签含义不同：OpenClaw是渲染图，Python是飞书API返回格式

### 统一格式标准（所有任务SOP）

所有8个cron任务的卡片格式已更新为：
```json
{
  "schema": "2.0",
  "header": {
    "title": {"tag": "plain_text", "content": "📋 标题 | YYYY-MM-DD"},
    "template": "orange"
  },
  "body": {
    "elements": [
      {"tag": "markdown", "content": "━━━ **一、章节标题** ━━━"},
      {"tag": "markdown", "content": "内容..."}
    ]
  }
}
```

**排版规则**：
- 章节标题：`━━━ **一、标题** ━━━`
- 编号列表：`1️⃣ 2️⃣ 3️⃣` 或 `① ② ③`
- 数字高亮：`**数字**`
- 风险/正面标注：`⚠️` / `🔥`
- 趋势符号：`↑ ↓ ➖`
