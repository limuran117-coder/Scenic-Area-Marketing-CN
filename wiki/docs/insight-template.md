# 洞察提炼模板

> 每周 INGEST 和临时洞察的统一格式要求 | 2026-04-28

---

## 标准格式

每条洞察必须包含以下4个部分，顺序不可调换：

```
## [洞察标题]

**数据事实：** （客观数字或现象，不带评价）
**业务意义：** （这个数据变化意味着什么）
**归因分析：** （可能的原因，可验证的假设）
**行动建议：** （具体的、可执行的建议）
**知识层归档：** （concepts/ 或 entities/ 中的具体路径）
```

---

## 示例

### 示例1：竞品动态洞察

```
## 万岁山武侠城搜索指数暴涨+41%

**数据事实：** 4月20日万岁山武侠城搜索指数196,492，日环比+41%，全竞品第一
**业务意义：** 万岁山热度大幅超越其他竞品，可能截流电影小镇潜在游客
**归因分析：** 「王婆说媒」持续出圈 + 武侠文化节活动，吸引大量自然流量
**行动建议：** 监测万岁山近期营销动作，考虑「错峰差异化」策略（避开直接竞争）
**知识层归档：** entities/万岁山武侠城.md → 热度分析章节
```

### 示例2：内容爆款洞察

```
## 「听劝标题」公式在小红书高转化

**数据事实：** 「避坑攻略」类标题笔记7.7万赞/12.7万藏，远超普通攻略类内容
**业务意义：** 实用型内容（帮助做决策）比展示型内容（单纯展示）转化率更高
**归因分析：** 用户决策前需要「避坑」，「听劝」感降低购买焦虑
**行动建议：** 电影小镇抖音/小红书内容增加「避坑」「建议」「攻略」类标题和实用信息
**知识层归档：** concepts/内容公式.md → 听劝标题
```

### 示例3：数据异常洞察

```
## Q1加购占比首次超过套票占比（⚠️预警）

**数据事实：** 2026年Q1加购占比50.5%，首次超过套票占比49.5%（2025年全年加购仅22.4%）
**业务意义：** 套票/加购结构可能发生根本性转变，客单价结构或将改变
**归因分析：** 可能原因①套票产品调整②游客自主加购意愿提升③五一前试探性消费
**行动建议：** 核实Q1是否有新票务组合上线，追踪4-5月结构是否持续
**知识层归档：** concepts/套票加购结构.md → Q1预警
```

## 洞察等级

| 等级 | 触发条件 | 处理方式 |
|------|---------|---------|
| 🔴 P0预警 | 数据异常可能影响核心KPI | 立即同步站长，当日处理 |
| 🟡 P1重要 | 重要趋势变化 | 加入周报，次日内处理 |
| 🟢 P2观察 | 值得留意但无需立即行动 | 写入知识层，下周跟进 |

---

## 禁止事项

- ❌ **禁止无数据依据的猜测**（必须基于实际数据）
- ❌ **禁止模糊归因**（"可能是..."不够，需有验证路径）
- ❌ **禁止重复已有结论**（先查知识层是否已有同类洞察）
- ❌ **禁止超出3句话的冗长描述**（长分析写进Wiki，不写进洞察卡片）

## 归档检查清单

写完洞察后自检：
- [ ] 数据来源是否标注（平台+日期）
- [ ] 业务意义是否明确（不是单纯描述数字）
- [ ] 行动建议是否具体可执行
- [ ] 是否已查知识层无重复
- [ ] 是否标注了归档路径
