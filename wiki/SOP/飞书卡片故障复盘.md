# 飞书卡片故障复盘

> 2026-04-19 | 故障时间：11:00-11:38

---

## 故障现象

- 抖音指数日报发送后，表格渲染异常（表格没有正确显示）
- 内容不完整，只有数据没有分析
- 飞书消息返回 `tag: "img"`（卡片内容被转成图片存储）

---

## 根本原因

### 1. OpenClaw card参数与飞书API不兼容

OpenClaw message tool的`card`参数传递方式与飞书IM API预期格式不匹配。飞书收到后把markdown卡片内容在服务端渲染成图片存储，API查询返回`tag: "img"`，这是**正常行为**，但导致后续调试时一直误判为"渲染失败"。

### 2. 分段方式错误

测试阶段把表格、亮点、洞察拆成多个`element`，导致渲染异常。

**错误做法：**
```json
"elements": [
  {"tag": "markdown", "content": "━━━ 竞品数据一览 ━━━"},
  {"tag": "markdown", "content": "| 表头...|"},   ← 表头和表格分开了
  {"tag": "markdown", "content": "| 表格行...|"}, ← 每行也分开了
]
```

**正确做法：** 每个分段内容完整独立，一个markdown元素承载一整块内容：
```json
"elements": [
  {"tag": "markdown", "content": "━━━ 竞品数据一览 ━━━"},
  {"tag": "markdown", "content": "| 景区 | 搜索指数 |...|\n|------|..."},  ← 表头+表格完整放一起
  {"tag": "markdown", "content": "━━━ 深度分析 ━━━\n\n🔥 ..."},  ← 整段分析独立一个element
]
```

---

## 解决方案

1. **放弃OpenClaw message tool的card参数**，改用Python脚本直调飞书IM API
2. 发送脚本：`scripts/send_feishu_card.py`
3. 卡片格式必须使用 `body.elements[]` 内嵌多个独立 `tag: "markdown"`
4. 发群前先在私信测试，确认格式OK再发大群

---

## 关键格式规范

```json
{
  "schema": "2.0",
  "header": {
    "title": {"tag": "plain_text", "content": "📊 抖音指数日报 | YYYY-MM-DD"},
    "template": "orange"
  },
  "body": {
    "elements": [
      {"tag": "markdown", "content": "━━━ 竞品数据一览 ━━━"},
      {"tag": "markdown", "content": "| 景区 | 搜索指数 | 搜索日环比 | 综合指数 | 综合日环比 |\n|------|---------|---------|---------|---------|"},   ← 表头+表格一起
      {"tag": "markdown", "content": "━━━ 建业电影小镇深度分析 ━━━\n\n🔥 ..."},
      {"tag": "markdown", "content": "━━━ 竞品格局分析 ━━━\n\n⚠️ ..."},
      {"tag": "markdown", "content": "━━━ 行动建议 ━━━\n\n① ..."},
      {"tag": "markdown", "content": "---\n*数据来源：抖音创作者平台 | 每周二更新*"}
    ]
  }
}
```

**规范：**
- 表头和表格必须放在**同一个markdown元素**里，不能拆开
- 亮点、洞察、分析各自独立成段
- 用 ↓↑ 符号（不用📉📈）
- 电影小镇行加粗 `**建业电影小镇**`，异动加 🔺
- 策略建议用 ①②③④ 编号
