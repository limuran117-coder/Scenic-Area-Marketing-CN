# 飞书卡片视觉规范

> 基于现有schema 2.0格式的视觉优化指南 | 2026-04-18确认

---

## 核心原则

- **不改变现有schema 2.0结构**（header + body.elements）
- **不出现代码块**（`````` ``` ```````）
- **不降级为纯文本消息**
- **每个模块之间用分隔线区分**
- **emoji作为视觉锚点，引导阅读顺序**

---

## 排版元素库

### 标题分隔线
```
━━━ 关键词 ━━━
```
> 用 `━━━` 而非 `---`，视觉更稳重

### 模块编号
```
🌿 01 | 标题文字
```
> emoji + 两位数编号 + `|` 分隔

### 要点列表
```
• 关键内容
• 关键内容（加粗：**高亮词**）
```

### 高亮数字
```
增长 **+68.6%**（数字加粗突出）
```

### 引用/强调
```
> 引用内容（少用，保持克制）
```

---

## 卡片结构模板

```
header.template: orange
header.title: 📊 日报类型 | YYYY-MM-DD

body.elements[0] — 大标题 + 分隔线
body.elements[1] — 模块一（含编号emoji）
body.elements[2] — 模块二
body.elements[3] — ……
body.elements[n] — 底部汇总/建议（用 ━━━ 分隔）
body.elements[n+1] — 数据来源（灰色小字）
```

## 视觉分级规则

| 级别 | 符号 | 用途 |
|------|------|------|
| 🔥 | 爆点/异动 | 重要异动、核心数据 |
| 📈📉➖ | 涨跌状态 | 趋势标注 |
| ⚠️ | 警示 | 风险、问题 |
| 💡 | 洞察 | 建议、分析 |
| 🎬🌿🌍📺 | 模块标识 | 不同类型内容区分 |
