## 通用格式标准（所有任务统一执行）

### JSON结构
```json
{
  "schema": "2.0",
  "header": {
    "title": {"tag": "plain_text", "content": "📋 标题 | YYYY-MM-DD"},
    "template": "orange"
  },
  "body": {
    "elements": [
      {"tag": "markdown", "content": "# 📌 一、章节标题\n\n内容..."},
      {"tag": "markdown", "content": "# 📌 二、章节标题\n\n内容..."}
    ]
  }
}
```

### 排版规则

| 元素 | 格式 | 示例 |
|------|------|------|
| 章节标题 | `# 📌 X、标题` | `# 📌 一、背景` |
| 编号列表 | `1️⃣ 2️⃣ 3️⃣` 或 `① ② ③` | `1️⃣ 内容...` |
| 要点 | `• 关键内容（**加粗**）` | `• 搜索指数 **4,925** ↑` |
| 数字高亮 | `**数字**` | `增长 **+68.6%**` |
| 风险标注 | `⚠️` | `⚠️ 风险提示` |
| 正面标注 | `🔥` | `🔥 增长亮点` |
| 趋势 | `↑ ↓ ➖` | `搜索 ↑ +14.32%` |
| 底部来源 | `---\n*数据来源：...*` | 放在卡片最底部 |

### 标题强化技巧（重要）
- **章节标题必须用 `#` Markdown标题标签**，这样飞书会渲染成更大更粗的标题
- 示例：`# 📌 一、背景` > `**一、背景**`（字号明显更大）
- 不要用纯加粗代替标题， `#` 标签是标题大小的关键

## 错误示范

❌ 用加粗代替标题
```json
{"tag": "markdown", "content": "**一、背景**\n内容..."}  // 字号和正文一样，不突出
```

❌ 表格拆成多个element
```json
{"tag": "markdown", "content": "| 表头..."},  // 表头和表格分开
{"tag": "markdown", "content": "| 数据行..."}   // 错误：应该放一起
```

❌ 没有章节分隔
```json
{"tag": "markdown", "content": "背景\n策略\n执行\n效果"}  // 没有分隔符，视觉混乱
```

## 正确示范

✅ 章节标题用 `#` 标签
```json
{"tag": "markdown", "content": "# 📌 一、背景\n\n• 要点1\n• 要点2"}
```

✅ 日报表格格式
```json
{"tag": "markdown", "content": "# 📌 竞品数据一览"},
{"tag": "markdown", "content": "| 景区 | 搜索指数 | 搜索日环比 | 综合指数 | 综合日环比 |\n|------|---------|---------|---------|---------|"}
```
