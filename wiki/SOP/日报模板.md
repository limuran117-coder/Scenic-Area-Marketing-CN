# 日报模板

> 每次日报输出后填写，便于追踪

## 格式规范（已确认，2026-04-19修正）

- header.title 用对象格式 `{"tag": "plain_text", "content": "标题"}`
- 内容放 `body.elements[]`（不是根级 elements[]）
- 需要 `schema: "2.0"`
- 标题用 **加粗**
- 表格：用 markdown 表格（tag: "markdown"），不用原生 table 元素
- 区块间用 "---" 或 "━━━" 分隔
- 数字压缩（157万而非1,571,756）
- 结论单独拎出，不堆在表格里
- 策略建议用 ①②③④⑤ 编号

## 抖音指数日报完整模板（2026-04-19确认）

**⚠️ 必须包含4个部分，缺一不可：**
1. 竞品数据表格（综合指数降序）
2. 建业电影小镇深度分析
3. 竞品格局分析
4. 行动建议（①②③④编号）

```json
{
  "schema": "2.0",
  "header": {
    "title": {"tag": "plain_text", "content": "📊 抖音指数日报 | 2026-04-19（数据周期近30天）"},
    "template": "orange"
  },
  "body": {
    "elements": [
      {"tag": "markdown", "content": "━━━ 竞品数据一览 ━━━"},
      {"tag": "markdown", "content": "| 景区 | 搜索指数 | 搜索日环比 | 综合指数 | 综合日环比 |\n|------|---------|---------|---------|---------|\n| 清明上河园 | 174,087 ↓ | -11.46% | 59,634 ↓ | -8.78% |\n...（8个景区，按综合指数降序，电影小镇加粗🔺）"},
      {"tag": "markdown", "content": "━━━ 建业电影小镇深度分析 ━━━\n\n🔥 **双指标增速双第一...**\n• 搜索指数 X,XXX，近7日涨幅 **+XX.XX%**（8个竞品中第一）\n• ...（分析要点3-4条）"},
      {"tag": "markdown", "content": "━━━ 竞品格局分析 ━━━\n\n⚠️ **头部松动**：...\n📈 **中部崛起**：...\n🔮 **黑马出现**：...\n\n**对电影小镇的启示**：..."},
      {"tag": "markdown", "content": "━━━ 行动建议 ━━━\n\n① **建议标题**：具体内容\n② **建议标题**：具体内容\n③ **建议标题**：具体内容\n④ **建议标题**：具体内容"},
      {"tag": "markdown", "content": "---\n*数据来源：抖音创作者平台 | 每周二更新*"}
    ]
  }
}
```

**关键点：**
- 表格分段和分析分段各自独立，不要合并
- ↓↑ 符号，异动用 🔺 标记
- 电影小镇行加粗 `**建业电影小镇**`
- 策略建议用 ①②③④ 编号

## 数据准确性原则（2026-04-15确认）

- **未确认不发送**：数据不完整时不发报告，先补全
- **禁止经验主义**：每项工作先读取全量原始数据，再分析判断
- **验证优先**：采集后两次验证一致性，确认再发群

## ⚠️ 静默执行规范（所有日报任务通用，禁止例外）

所有通过 cron announce 模式发送到群里的日报任务，**禁止**在群里输出操作过程。

**禁止出现在群里的内容**：
- 「正在执行xxx」「数据已采集」「操作完成」等过程性消息
- 「Let me check...」「Based on the data I collected...」等脚本内心白
- 中间步骤日志、抓取过程描述
- 把脚本执行流程发到群里

**只允许发送到群里的内容**：
- 提炼后的关键发现（数据+分析+建议）
- 符合 SOP 格式的飞书卡片

**操作状态记录方式**：只写在私信日志或 session 历史中，不发到群里
