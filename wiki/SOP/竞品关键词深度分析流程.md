## 二、四平台操作流程

### 平台1：抖音指数（核心平台）

**目标URL：** https://creator.douyin.com/creator-micro/creator-count/arithmetic-index

**CDP端口：** 18800（专属浏览器）

**执行步骤：**

**Step 1：用cdp_keyword_deep.py脚本采集（推荐）**
```bash
python3 ~/.openclaw/workspace/scripts/cdp_keyword_deep.py "关键词"
```
脚本会自动完成：搜索框输入→滚轮下滑→关联分析tab→人群分析tab

**必须采集的三个维度：**
- 搜索指数 + 综合指数（数值+日环比）
- 关联词TOP10（搜索上游词，反映用户决策路径）
- **人群画像（完整4字段：地域分布TOP10含TGI + 性别比例 + 年龄段分布 + TGI指数）⚠️不得缺失**

**Step 2：验证数据完整性**
检查返回数据是否包含：
- `related_words` 数组（至少5条）
- `crowd_profile.地域分布` 数组（至少10条）
- `crowd_profile.性别` 字段

### 平台2：小红书灵犀

**目标URL：** https://idea.xiaohongshu.com/idea/trend/trendAnalyze

**操作方式：** 打开专属浏览器Tab4（小红书灵犀）

**执行步骤：**
1. 在关键词搜索框输入目标关键词
2. 使用DOM注入脚本触发搜索（React组件普通type无效）：
```javascript
const inp = document.querySelector('input.d-text');
if (inp) { inp.focus(); inp.value = '关键词'; inp.dispatchEvent(new Event('input', {bubbles:true})); }
```
3. 等2秒后点击下拉选项
4. 等待数据加载，采集：
   - 搜索总量（万为单位）
   - 环比/同比变化
   - 热门搜索词TOP10
   - 搜索上下游词

**采集字段清单：**
| 字段 | 说明 | 示例 |
|------|------|------|
| 搜索总量 | 近7天搜索次数 | 12.5万 |
| 环比 | 较上周期变化 | +16.72% |
| 同比 | 较去年同期变化 | -8.70% |
| 热门搜索词 | 搜索量降序TOP10 | 银基动物王国攻略/门票/奇翼飞鹰等 |
| 上游词 | 用户搜索前的行为 | 去银基需要准备什么 |
| 下游词 | 用户搜索后的行为 | 银基住哪里好 |

## ⚠️ CDP浏览器操作规范（强制，禁止违反）

### 核心原则
**用已登录的Tab，不要新建空白页。**

错误做法：
```python
browser = await p.chromium.connect_over_cdp(CDP)
page = await browser.new_page()  # ❌ 这是空白页，没有登录态
await page.goto(url)  # ❌ 绕过了已登录的浏览器
```

正确做法（两步）：
```python
# 第一步：连接CDP
browser = await p.chromium.connect_over_cdp("http://127.0.0.1:18800")

# 第二步：列出已有Tab，用已登录的那个
for ctx in browser.contexts:
    for pg in ctx.pages:
        url = pg.url
        print(f"Tab: {url}")  # 根据URL判断哪个是哪个Tab
        
# 确定目标Tab后，直接操作，不需要goto
# 如果必须新建Tab：
new_pg = await ctx.new_page()  # ✅ 创建新Tab在同一个context里，有登录态
```

### 专属浏览器Tab对应关系
| Tab | URL | 用途 |
|-----|-----|------|
| Tab0 | 小红书灵犀 | 灵犀后台 |
| Tab2 | 抖音订阅页 | 订阅数据 |
| Tab4 | 抖音关键词页 | 关键词搜索（注意：需要登录态） |
| Tab6 | 小红书探索页 | 探索页搜索 |

### CDP连接后第一步
```python
browser = await p.chromium.connect_over_cdp("http://127.0.0.1:18800")
for ctx in browser.contexts:
    for i, pg in enumerate(ctx.pages):
        try:
            print(f"Tab{i}: {pg.url[:60]}")  # 确认目标Tab
        except:
            pass
```

### 常见错误
- `Frame has been detached`：page/frame在导航过程中被关闭 → 换一个Tab
- 空白登录页：新建了page而不是用已有Tab
- 登录态丢失：CDP连接到了错误的browser context

## 八、今日故障记录（2026-04-25）

### 故障1：竞品关键词深度分析老君山格式错误

**现象**：cron执行时发送了普通文本消息，而非飞书卡片格式

**根因**：cron agent的`announce`投递模式会直接发送文本，不会自动调用飞书卡片API

**正确做法**：
1. cron执行时必须调用`send_feishu_card.py`脚本发送卡片
2. 不能依赖`announce`模式直接发文本到群
3. 执行流程：`cdp_keyword_deep.py` → 采集数据 → 手动拼装JSON → `send_feishu_card.py`发卡片

**处理**：发送「抱歉格式有误撤回重发」说明消息，然后重新用卡片格式发送

---

### 故障2：xiaohongshu_crawl.py脚本无输出

**现象**：脚本执行后没有任何输出，超时结束

**可能原因**：
- 页面未完全加载，脚本等待超时
- 小红书反爬机制拦截
- Cookie失效

**处理方式**：
1. 先用CDP浏览器手动验证页面是否可正常访问
2. 检查Cookie是否有效（重新登录）
3. 作为临时方案：用CDP浏览器+手动操作采集小红书数据

**替代方案**：直接打开 `https://idea.xiaohongshu.com` 手动采集

---

### 故障3：百度搜索被反爬拦截

**现象**：`web_fetch`或`requests`访问百度返回安全验证页面

**原因**：百度有严格的反爬机制，无头请求会被拦截

**处理方式**：
1. 使用CDP浏览器（Tab3）打开百度URL
2. 等待页面完全加载后手动查看
3. 或直接使用已有案例库/竞品档案中的百度数据

---

### 故障4：飞书消息无法删除

**现象**：发送错误消息后，飞书API不支持删除已发送消息

**处理**：发送「抱歉格式有误撤回重发」说明消息，然后重新发送正确格式
