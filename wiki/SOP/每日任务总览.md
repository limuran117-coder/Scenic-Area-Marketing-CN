## 断点续做机制

竞品关键词cron和文旅营销案例cron共享状态文件：

```
状态文件：/tmp/daily_task_state.json
```

结构：
```json
{
  "竞品关键词": {
    "done": ["万岁山武侠城", "清明上河园", "只有河南戏剧幻城"],
    "current": "郑州方特欢乐世界"
  },
  "文旅案例": {
    "done": ["万岁山武侠城", "银基动物王国"],
    "current": "郑州方特欢乐世界"
  }
}
```

每次执行前读取done数组，完成后追加。

## 任务失败处理流程

```
cron error → 检查 lastError
├─ "timeout" → payload.timeoutSeconds 是否够用
├─ "no route" → delivery.mode 改为 none
├─ "message failed" → Python脚本发飞书失败 → 检查脚本日志
└─ "python error" → 查对应脚本

# 查看最近cron执行状态
openclaw tasks 2>&1 | grep -E "error|warn" | head -20
```
