## 技能一：knowledge-graph-memory（知识图谱记忆）

### 基本信息
- **位置**: `~/.openclaw/workspace/skills/jpeng-knowledge-graph-memory/SKILL.md`
- **核心功能**: 构建知识图谱 + 概念漂移检测 + 时序推理
- **作者**: jpeng

### 核心架构

**三大组件：**

1. **KnowledgeGraph** — 知识图谱
   - 节点(Node)：概念、实体、事件、事实、关系
   - 边(Edge)：IS_A/HAS_A/RELATED_TO/CAUSES/PRECEDES/INCLUDES/SIMILAR_TO/DERIVED_FROM
   - 支持路径查找(`findPath`)、关联查询(`getRelated`)、搜索(`search`)

2. **DriftDetector** — 概念漂移检测
   - 方法：ADWIN、DDM、统计方法
   - 用途：当某个概念的定义/属性随时间发生显著变化时自动告警
   - 场景：监控用户行为模式变化、市场趋势漂移等

3. **TemporalReasoner** — 时序推理
   - 按时间范围查询事件
   - 支持事件前/后查询
   - 用途：追踪知识演进历史

4. **Memory** — 记忆管理
   - 短时记忆 + 长时记忆自动巩固
   - 重要性评分 + 访问频率决定是否提升到长时记忆

### API 亮点
```javascript
kg.link('AI', 'Machine Learning', EdgeType.IS_A)  // 关系链接
kg.findPath('Deep Learning', 'Technology')        // 路径查找
kg.detectDrift('User Behavior')                    // 漂移检测
memory.consolidate()                              // 记忆巩固
```

### 应用场景
- 用户画像/行为模式的长期追踪
- 竞品数据的时间序列分析
- 电影小镇舆情/搜索趋势的知识积累

### 是否可集成到当前工作流？
**评估**: ⭐⭐⭐⭐ (4/5)
- 已有 memory/ 日报记忆系统，但缺乏图结构
- 可用于：将竞品数据、搜索指数关联起来形成知识网络
- 当前痛点：数据分散，缺乏关联分析能力
- **建议场景**：竞品数据每日的关联词、人群画像数据逐步沉淀为知识图谱

## 技能二：agent-swarm-ex（多智能体集群编排）

### 基本信息
- **位置**: `~/.openclaw/workspace/skills/agent-swarm-ex/SKILL.md`
- **核心功能**: 多智能体并行/串行编排完成复杂任务
- **适用场景**: 复杂项目、多角色协作、并行提效

### 核心架构

**10种专业智能体：**

| Agent ID | 角色 | 核心能力 |
|----------|------|----------|
| `pm` 📋 | 规划者 | 需求分析、任务拆解 |
| `researcher` 🔍 | 信息猎手 | 多源搜索、结构化输出 |
| `coder` 👨‍💻 | 代码工匠 | 编程调试 |
| `writer` ✍️ | 文字工匠 | 文档报告 |
| `designer` 🎨 | 视觉创作者 | 配图可视化 |
| `analyst` 📊 | 数据侦探 | 数据分析统计 |
| `reviewer` 🔎 | 质量守门人 | 代码/内容审核 |
| `assistant` 💬 | 沟通桥梁 | 消息转发 |
| `automator` 🤖 | 效率大师 | 自动化脚本 |

### 编排模式

**并行执行** — 多个子任务同时派发：
```javascript
sessions_spawn({ task: "搜索 LangChain...", agentId: "researcher", label: "r1" })
sessions_spawn({ task: "搜索 AutoGPT...", agentId: "researcher", label: "r2" })
sessions_spawn({ task: "搜索 CrewAI...", agentId: "researcher", label: "r3" })
// 三个任务并行执行，各自回报
```

**串行执行** — 等待前序完成再继续：
```javascript
// Phase 1: 并行调研
sessions_spawn({ task: "...", agentId: "researcher", label: "r1" })
sessions_spawn({ task: "...", agentId: "researcher", label: "r2" })
// 等待所有回报...

// Phase 2: 串行写作（基于汇总结果）
sessions_spawn({ task: "整合资料撰写报告...", agentId: "writer" })
```

### 编排示例
```
复杂任务 → 自动拆解 → 并行/串行执行 → 汇总交付

示例：竞品周报
├── 🔍 researcher × 5 (并行，5个竞品)
├── 📊 analyst (串行，汇总分析)
├── ✍️ writer (串行，撰写报告)
└── 🔎 reviewer (串行，质量审核)
```

### 应用场景
- **日报自动化**：并行采集5个竞品数据 → 串行分析 → 汇总输出
- **周度深度报告**：并行调研多个维度 → 专家分析 → 综合报告
- **大型市场调研**：并行覆盖多个信息源 → 整合分析

### 是否可集成到当前工作流？
**评估**: ⭐⭐⭐ (3/5)
- 当前任务已部分实现了类似功能（cron自动任务+子Agent派发）
- 优势：有标准化框架和统计模板
- 局限：需要配置多个模型，成本较高
- **建议场景**：季度深度报告、重大专题调研时启用，平时cron足够
