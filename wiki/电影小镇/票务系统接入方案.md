## 三、实时客流监控

### 3.1 核心指标

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| **实时在场** | 累计入园 - 累计出园 | 当前园区内人数 |
| **实时入园** | 当日已入园人次 | 含多次入园 |
| **实时出园** | 当日已出园人次 | 含多次出园 |
| **入园速度** | 近1小时入园人数 | 判断是否在峰值 |
| **平均停留时长** | 出园时间 - 入园时间均值 | 辅助判断游客行为 |

### 3.2 告警规则

| 场景 | 阈值 | 动作 |
|------|------|------|
| 在场人数 > 安全容量 | > 15,000人 | 🔴 告警：启动分流预案 |
| 在场人数 < 预期下限 | < 2,000人（节假日） | 🟡 告警：检查渠道是否有问题 |
| 入园速度骤降 | 较前1小时 < -50% | 🟡 告警：是否有突发情况 |
| 出园 > 入园（持续2h） | 出园-入园 > 5,000 | 🟡 告警：客流出逃，可能是体验问题 |

### 3.3 呈现形式

```
【实时客流】15:30
在园：8,237人（↑1,245较14:30）
今日入园：12,584人
今日出园：4,347人
入园速度：1,245人/h（近1小时）
峰值时段：11:00-13:00（3,200人/h）

⚠️ 15:00场余票仅剩47张，建议加开场次或启动营销引流
```

### 4.3 核心告警规则

| 场景 | 阈值 | 等级 | 动作 |
|------|------|------|------|
| **滞销告警** | 距开场2小时，余票>70% | 🔴 | 告警：建议启动营销引流或调整票价 |
| **爆卖告警** | 距开场1小时，余票<15% | 🟡 | 提示：可考虑加场或扩大座位 |
| **渠道失衡** | 单渠道占比>60% | 🟡 | 提示：某渠道异常集中或某渠道几乎无销售 |
| **退票异常** | 单场退票率>10% | 🔴 | 告警：可能有体验问题或系统问题 |
| **核销率低** | 开场后30分钟，核销率<60% | 🟡 | 提示：可能有人迟到/误场 |
| **早场票积压** | 早场开售12小时，上座率<30% | 🟡 | 提示：早场票销售不畅 |

### 4.4 预测模型

基于当前售票速度，预测每场次售罄时间：

```
公式：预计售罄时间 = 当前时间 + (余票量 / 最近2小时售票速度)

示例（5月1日 15:30场）：
  当前时间：15:00
  余票：173张
  最近2小时速度：62张/h
  预计售罄：15:00 + 173/62 ≈ 17:47

  ⚠️ 距开场仅47分钟，余票173张
  建议：立即检查各渠道余票，协调窗口现场引导
```

## 五、退票与异常监控

### 5.1 退票监控

| 指标 | 计算 | 告警阈值 |
|------|------|---------|
| 退票率 | 退票量 / 总售出量 | > 10% 触发检查 |
| 集中退票 | 某场次1小时内>30张退票 | 触发根因分析 |
| 退票来源 | 窗口/线上各占比例 | 线上异常偏高提示系统问题 |

### 5.2 核销异常

| 场景 | 说明 |
|------|------|
| **未核销预警** | 开场后1小时仍有大量票未核销，可能观众误场 |
| **超售预警** | 核销量 > 座位数（数据异常） |
| **重复核销** | 同一订单被核销多次（系统bug或黄牛） |

### 5.3 数据一致性校验

```
实时校验：已售 + 已退 + 已核销 = 总票量

若不等，触发 🔴 数据异常告警
→ 可能原因：系统延迟/数据同步问题/黑屏问题
→ 动作：立即检查票务系统接口状态
```
