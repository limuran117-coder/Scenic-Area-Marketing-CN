## 二、核心异常场景（旅游景区版）

### 2.1 客流类异常

旅游景区异常的核心是**"突然超出预期"**，而非退票。

| 异常类型 | 触发条件 | 风险等级 | 说明 |
|---------|---------|---------|------|
| **瞬时入园激增** | 某15分钟窗口入园量 > 平均值3倍 | 🔴 | 入口排队堆积，安全风险 |
| **入园速度骤降** | 某15分钟窗口入园量 < 平均值30% | 🔴 | 入口闸机故障/通道关闭 |
| **在园人数超容量** | 实时在场 > 安全容量 | 🔴 | 启动分流/限流预案 |
| **持续净流入** | 出园 < 入园连续4小时 | 🟡 | 客流量持续积累，需关注出口引导 |
| **出园陡增** | 某时段出园量 > 入园量+均值的2倍 | 🟡 | 可能是大团队离场或体验问题 |
| **入园数据断流** | API数据超过30分钟无更新 | 🔴 | 数据采集中断，可能系统异常 |

### 2.2 演出票类异常

| 异常类型 | 触发条件 | 风险等级 | 说明 |
|---------|---------|---------|------|
| **某场次滞销** | 距开场2小时，余票>70% | 🔴 | 错过引流时机，场次空置 |
| **某场次爆卖** | 某渠道余票<5%但其他渠道充裕 | 🟡 | 单一渠道超发或系统同步延迟 |
| **渠道完全无销售** | 某渠道持续30分钟零订单 | 🟡 | 该渠道系统故障或接口异常 |
| **全场销售停滞** | 所有场次连续1小时无新订单 | 🔴 | 票务系统接口可能中断 |
| **早场票滞销** | 早场开售12小时，上座率<30% | 🟡 | 早场定位问题，需调整定价或营销 |

### 2.3 数据一致性异常

| 异常类型 | 触发条件 | 风险等级 | 说明 |
|---------|---------|---------|------|
| **数据校验失败** | 已售+已核销 ≠ 总入场人数 | 🔴 | 系统数据异常，需立即核查 |
| **核销>售出** | 核销量 > 订单量 | 🔴 | 系统bug或数据同步问题 |
| **实时数据跳变** | 某指标5分钟内变化 > 50%（非渐进） | 🔴 | 接口数据抖动，可能是系统问题 |

## 三、实时客流监控

### 3.1 核心指标

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| **实时在场** | 累计入园 - 累计出园 | 当前园区内人数 |
| **实时入园** | 当日已入园人次 | 含多次入园 |
| **实时出园** | 当日已出园人次 | 含多次出园 |
| **入园速度** | 近15分钟入园人数（滚动） | 捕捉瞬时激增 |
| **出园速度** | 近15分钟出园人数（滚动） | 判断客流去向 |

### 3.2 告警规则

| 场景 | 阈值 | 等级 | 动作 |
|------|------|------|------|
| 瞬时入园激增 | 某15分钟 > 平均3倍 | 🔴 | 立即告警：入口排队/启动分流广播 |
| 入园速度骤降 | 某15分钟 < 平均30% | 🔴 | 立即告警：检查闸机/通道是否正常 |
| 在场人数超容量 | > 安全容量 | 🔴 | 启动限流预案，停止窗口售票 |
| 出园陡增 | 出园>入园+2倍均值 | 🟡 | 关注是否有体验投诉堆积 |
| 数据断流 | API无响应>30分钟 | 🔴 | 切换备用数据源，人工确认系统状态 |
| 数据跳变 | 5分钟内变化>50% | 🔴 | 暂停推送，人工核查数据来源 |

### 3.3 推送格式示例

```
【实时客流】5月1日 11:00

园区状态：
  在园：6,342人（安全容量内）
  今日入园：9,847人
  今日出园：3,505人
  入园速度：892人/15min（正常，峰值约1200）
  出园速度：234人/15min

《穿越德化街》场次状态：
  ✅ 10:00场 已售罄（100%）
  ✅ 11:40场 已售罄（98%）
  🟡 13:30场 剩余287张/1,140（25%），距开场2.5小时
  ⚠️ 15:30场 剩余876张（77%），销售偏慢，关注滞销风险
  ✅ 17:10场 剩余1,003张（88%），正常

接口状态：✅ 票务API正常，数据延迟<1分钟
```

### 4.3 预测模型

```
预计售罄时间 = 当前时间 + (余票量 / 最近2小时平均售票速度)

示例（5月1日 15:30场）：
  当前：15:00 | 余票：173张 | 速度：62张/h
  预计售罄：15:00 + 173/62 ≈ 17:47
  距开场仅47分钟，余票173张
  ⚠️ 建议：立即协调窗口现场引导
```

## 五、数据一致性校验

每次采集数据时，实时校验：

```
已售 + 已核销 = 总入场人数？

若不等 → 🔴 数据异常告警
可能原因：
  ① 系统延迟（数据写入滞后）
  ② 闸机与票务系统不同步
  ③ 数据接口存在bug
处理：立即切换备用数据源，人工确认
```

## 七、技术实现

### 7.1 轮询策略

| 时段 | 频率 | 说明 |
|------|------|------|
| 演出开场前2小时 | 每15分钟 | 高频，捕捉滞销/爆卖 |
| 演出开场后1小时 | 每30分钟 | 核销追踪 |
| 其他时段 | 每30分钟 | 客流监控 |
| 数据断流后 | 每5分钟 | 持续探测直到恢复 |

### 7.2 异常处理流程

```
数据采集异常
    ↓
触发数据断流告警（>30分钟无响应）
    ↓
切换备用数据源（若配置了）
    ↓
人工介入检查（通知站长）
    ↓
系统恢复后补录数据
    ↓
异常结束归档（wiki/异常记录/）
```
